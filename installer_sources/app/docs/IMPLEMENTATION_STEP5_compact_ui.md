# Реализация — Компактная визуализация и уплотнение интерфейса (6 марта 2026)

## Статус: Выполнено

## Краткое содержание

**Цель:** Сделать страницу «Проекты и этапы» и связанные экраны более компактными, улучшить читаемость названий проектов на диаграмме Ганта, исправить ошибку отображения этапов при иерархии.

**Сделано:**

1. **Страница «Проекты и этапы»**
   - Внутри раскрытой карточки проекта — крупный заголовок с названием и подпись «компания • статус».
   - Этапы выводятся в порядке дерева (phases_tree_order); основные этапы (depth=0) — жирным с нумерацией «1.», «2.»; подэтапы — с маркером «•» и отступом по уровню (16px на уровень).
   - Назначенные сотрудники перенесены в отдельную колонку: метка «👥 N» или «Наслед.»; по клику — popover (Streamlit 1.32+) или expander со списком сотрудников.
   - Тип, даты и готовность в списке этапов выводятся через `st.caption` для уменьшения вертикальных отступов.
   - Форма редактирования этапа — три колонки (название/тип; даты; готовность), подписи через caption; форма добавления этапа — три колонки, кнопка «Добавить этап» с `use_container_width=True`.
   - Локальный CSS для страницы проектов (`_render_projects_compact_css`) — уплотнение expander, форм и отступов внутри карточек.

2. **Блок «➕ Новый проект»**
   - Expander по умолчанию свёрнут (`expanded=False`).
   - Форма в трёх колонках: название и компания; ведущий, загрузка и статус; авто-даты и при выключении — даты начала/окончания. Секция «Рядовые сотрудники» вынесена во вложенный свёрнутый expander. Кнопка «Создать проект» с `use_container_width=True`.

3. **Глобальный CSS (`.streamlit/styles.css`)**
   - Уменьшены вертикальные отступы между блоками (margin-bottom 0.5rem), отступы у форм и expander, компактнее кнопки. Изменения применяются глобально ко всем страницам.

4. **Диаграмма Ганта**
   - Подписи проектов на оси Y в формате «ПРОЕКТ: название» для лучшей читаемости.
   - Размер шрифта подписей оси Y — 11; левый отступ диаграммы задаётся адаптивно по максимальной длине подписи (90–120 px).
   - Подписи этапов в формате «• название этапа (название проекта)».

5. **Исправление KeyError 'id'**
   - В `repository.phases_tree_order` и `update_parent_phase_dates_from_children` используется `set_index("id", drop=False)`, чтобы колонка `id` оставалась в строках DataFrame при обращении `ph['id']` в UI.
