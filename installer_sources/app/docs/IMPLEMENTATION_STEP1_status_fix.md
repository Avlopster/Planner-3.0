# Реализация плана — Пункт 1: Сохранение статуса проекта в БД

## Статус: Выполнено

## Краткое содержание

**Проблема:** При смене статуса проекта на странице «Проекты и этапы» (кнопка «Сохранить статус») значение записывалось в файл БД через `update_project_status_on_disk`, но после перезагрузки страницы отображался старый статус. Причина — при `st.rerun()` создавалось новое подключение к БД через `get_connection(_initial_db_path)` (путь из конфига), а не к фактически использовавшемуся пути (например fallback `%LOCALAPPDATA%\Planner\resource_planner.db`). В результате интерфейс показывал данные из «конфиговой» БД, где статус не обновлялся.

**Решение:** При создании подключения к БД использовать ранее сохранённый путь из `st.session_state["_db_path"]`, если он есть, чтобы после сохранения статуса и rerun приложение подключалось к тому же файлу.

## Изменённые файлы

- **Planner 3.0/Planner.py** — в блоке инициализации подключения (строки 18–21): перед вызовом `db.get_connection` вычисляется `path_to_use = st.session_state.get("_db_path", _initial_db_path)`, вызов заменён на `db.get_connection(path_to_use)`.
- **Planner 3.0/installer_sources/app/Planner.py** — аналогичное изменение для копии точки входа установщика.

## Проверка

После смены статуса на «Планируется» и обновления страницы статус должен остаться «Планируется». Фильтр «Только активные проекты (В работе)» на диаграмме Ганта должен корректно исключать такие проекты.
