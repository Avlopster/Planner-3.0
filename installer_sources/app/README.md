# Planner 3.0

Ветка разработки многопользовательского режима Planner. Основа — копия приложения Planner 2.0; вся новая функциональность (авторизация, аудит пользователей, поддержка PostgreSQL, серверное развёртывание) реализуется здесь, без изменений в основном проекте Planner 2.0.

## Назначение

- Серверная установка (Windows / Linux)
- Система авторизации пользователей
- Производительная СУБД (SQLite по умолчанию, опционально PostgreSQL)
- Логирование действий пользователей (аудит)

## Запуск

```bash
cd "Planner 3.0"
pip install -r requirements.txt
streamlit run Planner.py
```

Приложение будет доступно по адресу http://localhost:8501.

## Структура проекта и архитектура

- **Точка входа:** `Planner.py` — инициализация логов, подключение к БД, схема и миграции, боковое меню и роутинг по разделам. Не содержит кеша и обёрток: страницы работают с данными напрямую.
- **Страницы (UI):** `app_pages/*.py` — каждая страница экспортирует `render(conn)` и при необходимости `render(conn, db_path=...)`. Вызывают `repository`, `load_calculator`, `capacity` и используют `components` для общих виджетов.
- **Данные:** `database.py` — подключение (`get_connection` возвращает `(conn, actual_path)`), схема, миграции. `repository.py` — все загрузки и изменения данных (CRUD проектов, этапов, назначений, конфиг и т.д.); только параметризованные запросы.
- **Бизнес-логика:** `load_calculator.py` (загрузка по дням, сводки, перегрузки), `capacity.py` (годовая ёмкость проектов). Описания правил — в [logic.md](logic.md).
- **Безопасность:** страница «SQL-запросы» по умолчанию разрешает только чтение (SELECT, PRAGMA); для запросов на запись требуется явное подтверждение пользователя. Текст запроса не логируется.

Подробности рефакторинга и чек-лист этапов — в [docs/REFACTORING_STAGES.md](docs/REFACTORING_STAGES.md) и [docs/REFACTORING_RESULTS.md](docs/REFACTORING_RESULTS.md).

## Установка на Windows

В каталоге `installer/` подготовлен установщик Inno Setup с поддержкой **разностного обновления**: при повторной установке сохраняются конфигурация, БД и настройки; обновляются только файлы приложения и зависимости. См. [installer/README.md](installer/README.md).

## Изоляция от Planner 2.0

Данный проект имеет отдельный Git-репозиторий и не связан с репозиторием родительского проекта Planner. Изменения в «Planner 3.0» не затрагивают код и данные Planner 2.0.
