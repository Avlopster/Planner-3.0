# Workflow: доработка интерфейса по тасклисту

Источник задач: [New_interface.md](../New_interface.md).

---

## Правила для кодового ассистента

1. **Работать строго по плану** — только задачи из New_interface.md, в порядке итераций 1→5.

2. **Перед каждой итерацией** — предложить решение и **согласовать с пользователем** (при необходимости — с указанием файлов и фрагментов кода). Не приступать к правкам до согласования.

3. **После согласования** — реализовать изменения в коде.

4. **После реализации** — дождаться **подтверждения** пользователя (проверка в браузере / ревью).

5. **Обновить прогресс:**
   - в New_interface.md поставить галочки `- [x]` у выполненных пунктов итерации;
   - в таблице «Отчёт по прогрессу» сменить статус итерации на 🟡 (В работе) или ✅ (Готово), при завершении — указать дату.

6. **Переход к следующей итерации** — только после явного согласования с пользователем.

7. **Коммит** — после завершения итерации и обновления прогресса сделать коммит в репозиторий (осмысленное сообщение, например: `UI: итерация 1 — тема Streamlit`).

---

## Цикл по одной итерации

```
Согласовать решение (и при необходимости фрагменты кода)
    → Реализовать
    → Ждать подтверждения
    → Отметить задачи и обновить отчёт в New_interface.md
    → Согласовать переход к следующей итерации
    → Коммит
    → Следующая итерация
```
